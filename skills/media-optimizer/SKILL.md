**"让你的 AI 员工 24/7 自动运营自媒体"**

本 skill 是**编排调度层**，自身负责内容生成和数据分析，评估和发布委托给专业 skill：

- 📊 **数据驱动** - 基于数据反馈持续优化
- 🌐 **多平台协同** - 一套内容，/media-eval 自动适配五大平台

## 🛠️ 核心功能模块

### 1️⃣ 内容引擎

#### 素材采集
**来源：**
- 🔍 **热点监控** - 各平台热榜、热搜
- 📰 **知识库扫描** - Obsidian 笔记自动提取
- 💡 **灵感库** - Inbox 中的想法
- 🤖 **AI 生成** - 基于历史数据生成新内容

#### 内容生成
**生成模式：**
1. **原创创作** - 基于素材库生成新内容
2. **内容重组** - 旧内容新包装
3. **A/B 测试** - 同一主题多个版本

**内容类型：**
- 教程类（How-to）
- 观点类（Opinion）
- 资讯类（News）
- 工具类（Tools）
- 故事类（Story）

---

### 2️⃣ 评估引擎（委托 /media-eval）

内容生成后，调用 `/media-eval` 进行专业评估和多平台改写。

#### 评估流程

```
生成原始内容
  ↓
调用 /media-eval 评估 + 生成各平台优化版本
  ↓
评分 >= 75（A级）──→ 进入发布环节
  ↓
评分 < 75 ──→ 将 /media-eval 的改写建议作为输入，再次调用 /media-eval 优化
  ↓
（最多循环优化 2 次）
  ↓
2 次后无论评分多少 ──→ 直接进入发布环节
```

#### 调用方式

**首次评估：**
```
/media-eval --platform tweet,jike,xhs,zhihu,douyin <生成的内容>
```

**优化改写（评分 < 75 时）：**
```
/media-eval --platform tweet,jike,xhs,zhihu,douyin <根据评估建议优化后的内容>
```

#### 评估输出利用

`/media-eval` 会输出：
1. **评估报告** — 内容本质层评分 + 平台适配度评分
2. **各平台优化版本** — Tweet/即刻/小红书/知乎/抖音 五个版本

发布环节直接使用 `/media-eval` 输出的各平台优化版本，不再自行适配。

---

### 3️⃣ 发布引擎（委托平台 Skill）

评估通过后，**同时**调用五个平台 skill 发布内容。

#### 支持平台

| 平台 | 发布 Skill | 风格 | 特点 |
|------|-----------|------|------|
| **Tweet (X)** | `/tweet` | 短平快 | 话题标签 + @提及，自动拆分 thread |
| **即刻** | `/jike` | 社区化 | 观点 + 互动，支持指定圈子 |
| **小红书** | `/xhs` | 种草风 | 自动文字转图片，支持话题标签 |
| **知乎** | `/zhihu` | 深度专业 | 支持 Markdown，可指定话题 |
| **抖音** | `/douyin` | 图文/视频 | 支持多图上传，可指定话题 |

#### 发布流程

```
/media-eval 输出的各平台优化版本
  ↓
同时调用（并行执行）：
├─ /tweet <Tweet 版本内容>
├─ /jike <即刻版本内容>
├─ /xhs <小红书版本内容>
├─ /zhihu <知乎版本内容>
└─ /douyin <抖音版本内容>
  ↓
记录各平台发布链接到 已发布.md（格式：| 日期 | 平台 | 标题 | 链接 | 评分 |）
```

#### 互动管理
**自动回复：**
- 关键词触发回复
- 欢迎新关注者
- 常见问题 FAQ

**智能互动：**
- 点赞高质量评论
- 关注潜力用户
- 收藏有价值内容

## 🎯 工作流程

### 完整运营流程（单次内容）

```
1. 素材采集（Obsidian 扫描 / 热点监控）
   ↓
2. 内容生成（原创 / 重组 / A/B 测试）
   ↓
3. /media-eval 评估 + 多平台改写
   ↓
4. 评分 < 75？→ /media-eval 优化（最多 2 次）→ 直接发布
   评分 >= 75？→ 进入发布
   ↓
5. 同时调用 /tweet /jike /xhs /zhihu /douyin 发布
   ↓
6. 记录发布链接，开始数据监控
```

### 每日运营流程

```
08:00 - 早会（AI 主动）
├─ 检查夜间数据

├─ 生成今日任务清单
└─ 发送报告给你

09:00 - 内容创作
├─ 基于热点生成内容
├─ 从队列选取待发布
├─ /media-eval 评估 + 改写
└─ 进入发布队列

12:00 - 午间发布
├─ 同时发布到五大平台
├─ 记录发布链接
└─ 初步监控反馈

18:00 - 晚间发布
├─ 发布第二轮内容
├─ 互动（回复/点赞）
└─ 数据更新

21:00 - 深度互动
├─ 重点评论深度回复

├─ 生成今日数据报告
├─ 优化明日计划

```

### 热点响应流程

```
监控触发 → 快速判断 → 内容生成 → /media-eval 评估一次 → 无论评分直接发布五平台
   ↓           ↓          ↓              ↓                       ↓
实时监控   相关性?   15分钟内      评估+改写（仅1次）       立即发布
各平台     >80%?     生成内容      不进入优化循环            追流量
```

> **热点场景特殊规则**：调用 /media-eval 评估一次获取各平台优化版本，无论评分高低直接发布，不进入"<75 优化循环"。热点追速度，常规追质量。

## 🤖 AI 主动性设计

### 主动触发场景

**高主动性（需要通知你）：**
- 🚨 突发热点（相关性 > 80%）
- 📉 数据异常（掉粉 > 10%）
- 💼 商业机会（合作邀请）
- ⚠️ 负面舆情（差评、投诉）

**中主动性（记录即可）：**
- 📊 常规数据更新
- 📝 内容队列更新
- 🔄 定时任务执行

**低主动性（静默执行）：**
- ✅ 点赞、收藏
- 👥 关注用户
- 📈 数据统计

### 决策框架

**"能不能做"判断：**
```
IF 事情在授权范围内
AND 符合运营策略
AND 风险可控
THEN 自动执行
ELSE 请求用户确认
```

**示例：**
```
场景：发现热点话题
判断：
1. 相关性 > 80%? YES
2. 符合独立开发定位? YES
3. 有商业风险? NO
→ 生成内容 → /media-eval 评估 → 同时发布五平台

场景：收到合作邀请
判断：
1. 在授权范围? NO (涉及商业)
2. 需要用户评估
→ 转发给用户，不做决定
```
